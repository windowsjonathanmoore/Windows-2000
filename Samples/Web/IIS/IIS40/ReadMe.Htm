Internet Information Server Platform SDK

Setup

1) Downloading & Installing

If you are reading this file from the Microsoft Platform SDK CD and have not already installed Microsoft Internet Information Server (IIS) 4.0 and created the correct directory structure, the samples included will not work. Some samples will work if you have IIS 3.0 installed; however, this IIS SDK is designed to illustrate the extended functionality of IIS 4.0. It is recommended that you use IIS 4.0 to run these samples. You can download and install Microsoft Windows NT® 4.0 Option Pack which contains IIS 4.0 from the Microsoft IIS Website.

2) Directory structure

Once you have installed IIS 4.0, you may need to create the correct directory structure in order for the samples to run correctly. The SDK samples on the Platform SDK CD supercede the ones that come with IIS 4.0 Beta 3. If you have this previous version of IIS, you should copy the contents of the (%mssdk%)/samples/internet/iis directory on the Platform SDK CD into the /your-wwwroot/iissamples/sdk directory (the default is Inetpub). Replace any files that already exist in the latter directory. If you install IIS 4.0 (release version), you should use the SDK found in the product itself. The virtual path that all the samples use is http://servername/iissamples/sdk. Make sure the permission setting for this directory is Execute. This is necessary for some samples to execute properly.

NOTE: If you install Windows NT 4.0 Option Pack and choose to install the SDK for IIS 4.0, the directory structure will be set up for you automatically. The SDK, however, is not installed as a part of IIS 4.0 by default. You will need to do a custom install and choose SDK under Internet Information Server\Documentation\SDK when prompted for components to install.

Samples

The samples are designed to expose you to the power and flexibility of IIS. ASP scripts, Active Server components, and ISAPI extensions and filters are included in the samples provided. Each section demonstrates both basic concepts and advanced techniques. The samples are organized as follows:
  • Admin -- This section contains Windows Scripting Host (WSH) samples that illustrate how to administer IIS using the WSH environment.
    • Application Creator
    • Document Footer Utility
    • Logging Module Enumerator
    • Metabase Backup Utility
    • Metabase Backup Enumerator
    • Metabase Backup Restore Utility
    • Metabase Backup Delete Utility
    • Web Server Creator

  • ASP -- This section contains sample scripts ranging in difficulty.

    • Simple -- These scripts illustrate the fundamental techniques used in ASP scripting. If you're a newcomer to application development, or you're a programmer who has never scripted before, this is a good starting place.
      • Variables
      • Looping
      • Conditional Operators
      • Arrays
      • Server-Side Includes
      • Functions and Procedures
      • Using Components

    • Applications -- With the tools and services provided with ASP, you can create an integrated and powerful application with a minimal amount of time and effort. Here you will find examples that show some of the basic application features in action.
      • Session Variables
      • Application Variables
      • Session Management

    • Interaction -- These examples demonstrate some of the techniques used in creating an interactive ASP application.
      • User Form Input with GET
      • User Form Input with POST
      • HTTP Server Variables
      • Using Cookies
      • Redirection
      • Setting Expiration Info
      • Setting PICS Ratings
      • Setting Content Type
      • Client Connection
      • Client-Side Scripting
      • Populating Fields
      • Login Screens
      • Response Buffering
      • CDF

    • Components -- To further enhance the power of ASP scripting, there are a number of installable components that your application can call upon to perform a wide variety of tasks.       • File Access

      • Content Rotator

      • Tools
      • Index

      • Query to Fill HTML Table

      • Basic Transaction

  • ISAPI -- Developers can add to the functionality of IIS using ISAPI and create Web applications that have as much functionality as any desktop application. ISAPI programs are divided into two types.

    • Extensions -- Extensions are loaded on-demand, to provide extended functionality to a Web application. The samples provided here range from the basic extension outline to a complex ISAPI program that deals with threads and keep-alives.
      • Simple ISAPI Extension
      • ISAPI and COM
      • Keep-Alive
      • Worker Thread
      • Keep-Alive with Worker Thread
      • Server Variables
      • Redirection Extension
      • Synchronous and Asynchronous I/O
      • ISAPI and Forms
      • CGI Wrapper
      • Chunked Transfer Encoding

    • Filters -- Filters are loaded with the server, and can be used to customize and enhance the services provided by the server. The samples included here demonstrate how to create and use your own ISAPI filters.
      • Uppercase Conversion Filter
      • Cookie Conversion Filter
      • Authentication Filter

      Important: Before a filter can be used, it must be registered with the Internet Service Manager. To register a filter,
      1. Go to the Start/Programs/Windows NT 4.0 Option Pack/Microsoft Internet Information Server/Internet Service Manager.
      2. Click with the right mouse button to choose Properties on the web site you want to use the filter in.
      3. Click the ISAPI FIlter tab.
      4. Click Add.
      5. Give the filter a name and point it to the correct file.
      6. Stop and restart the web site.
