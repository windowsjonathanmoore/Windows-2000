Technical Issues

Odbcsql Sample - Command Line

The Odbcsql Sample implements an ODBC command-line interpreter that allows you to use SQL commands and view the results from the selected data source. The sample is written as an ODBC 2.x application.

After allocating the environment and connection, the application connects to the driver specified or, if none, prompts you for the DSN. The command-line interpreter then prompts you for SQL commands to run against the data source.

Results are displayed in columnar form after memory is allocated for the buffers and column titles are written. Column information is fetched and bindings are allocated for each column. (The char data type is assumed when determining display length although you might have better performance with other types.) 

A buffer is allocated for the data and mapped to the driver's buffer, which will be used to hold the data at fetch time.

An error handling routine is also included.

Setup

Build the sample using the source file, Odbcsql.c, provided in the \Samples\ODBC\odbcsql directory. Determine the DSN to use.

Running the Sample

Run the executable file after successfully building the project. The sample will prompt you for a DSN. Once you are connected, the command-line prompt will appear. You can type SQL commands, evaluate the results, and compare them to the sample code.
