Welcome to Microsoft Host Integration Server 2000 WMI Test Page!

Microsoft Host Integration Server 2000 (Babylon) provides enhanced management capabilities for SNA Administrators to view and control their SNA Server environment. Using WMI/COM, an automation interface allows customers to derive their own admin tools that can be used to monitor, configure, and take action on all SNA admin objects. These example ASP pages show you how to configure a Web Site that will allow you to monitor status, start and stop services and connections, and make changes to the Host Integration Server without using the Administrator program.

These new features can be expanded upon with VBScript, JavaScript, C/C++, Visual Basic, and other programming languages.

 

Vision

We are all about management and manageability. 

The WMI vision is to enable the windows platform for monitoring, controlling, and analyzing software, hardware, networks, users, and policies within the enterprise.

WMI provides the plumbing or infrastructure for developers and ISVs to create management applications built directly on top of the managment infrastructure.

WMI is a windows product - we shipped infrastructure for Win95, Win98, NT4, and Windows 2000. WMI makes Windows 2000 the best solution for managing any platform and devices, including ones running in non-windows environments.

WMI enables the understanding of what is currently happening, changing what is currently happening, and determining what has already happened, for software applications, hardware components, and networks. This is built on the industry-standard CIM from the DMTF.

The managed environment extends beyond hardware and software to Users and Policies.

Details

The sample pages here demonstrate the following techniques:

3270 LUs:

  • MsSna_LuDisplay.asp - shows how to gather information on all 3270 Display LUs on the SNA Server

  • MsSna_LuLua.asp - shows how to gather information on all LUA LUs on the SNA Server

  • MsSna_LuPrint.asp - shows how to gather information on all 3270 Print LUs on the SNA Server

Adapters:

  • MsSNA_Adapter_Exec.asp - shows how to gather information on all installed adapters on the SNA Server. This file use a WQL Query (Call ExecuteQuery( WmiNameSpace , strQuery)) to gather the information. This sample also uses a different style to display the information to the user.

  • MsSNA_Adapter_Get.asp - this does the same as All_SNA_Adapters.asp, but uses WmiNamespace.Get("SNA_Adapter") to gather the information.

  • MsSna_AdapterToMachine.asp - this is a reference query that returns PathtoAdapter and PathToConnection

Make Changes:

  • Add_User.asp - this is a HTML/ASP wizard that allows the administrator to add users to his Host Integration Server environment without having to startup Host Integration Server's management console. This also demonstrates queries against NT to gather the users and groups installed on NT.

  • Delete_User.asp - This wizard demonstrates deleting a user from within SNA Server

  • Add_3270LUtoUser.asp - This wizard adds a 3270 Display LU to a user's record.

APPC:

  • MsSNA_AppcMode..asp - gathers mode information for all installed modes

  • MsSNA_AppcPartner.asp - gathers LU-LU-MODE partner information for any partnerships setup.

  • MsSna_LuAppcLocal.asp - gathers Local LU Information

  • MsSna_LuAppcRemote.asp - gathers Remote LU information.

Connections:

  • MsSna_Connection8022Dlc.asp - displays connection information and status for each 802.2 connection. This page also will allow an administrator to stop and start connections.

Domain:

  • MsSna_Domain.asp - gathers information about the SNA Server Domain.

Pools:

  • MsSna_Pool.asp - display pool information

Server:

  • MsSna_Server.asp - display node status for each installed Host Integration Server node

Services:

  • MsSna_ServicePrint.asp - display the SNA Print service configuration and status

  • MsSna_ServiceSharedFolder.asp - display SFGW status and configuration

  • MsSna_ServiceSna.asp - display SNA Server's snaservice status and information

  • MsSna_ServiceTN3270.asp - displays TN3270 configuration and status

  • MsSna_ServiceTN5250.asp - display TN5250 configuration and status

SFGW:

  • MsSna_AS400.asp - This page display AS/400 Shared Folder Definition

  • MsSna_AS400Folder.asp - This page displays AS/400 Shared Folder information

Status:

  • All_SNA_Services_Status.asp - this is a master page that monitors all installed SNA Services and connections, updating every 10 seconds. An administrator can monitor this page and stop and start services as needed.

Users:

  • MsSna_ConfiguredUser.asp - display configured user information.
