Installing MSMQ Triggers

MSMQ Triggers runs on Windows 2000 with Message Queuing installed and Windows NT 4.0 with Service Pack 4 and on MSMQ 1.0 installed (Intel-based processors only).

MSMQ Triggers cannot be installed on Windows 95, on Windows 98, on dependent client computers, or on server cluster nodes.

MSMQ Triggers is not installed by Windows 2000 Resource Kit Setup. To install it, use the following procedure:

To install MSMQ Triggers

  1. On the Resource Kit CD, open the x86\Apps\Msmqtriggers folder, and then double-click MSMQTriggers.exe to start the setup program. Follow the instructions that appear on the screen.
  2. If needed, configure the MSMQ Triggers Service to run under a domain administrative account for the local computer. The default LocalSystem account can also be used to run this service.
  3. Grant the service account the Peek Message permission for all queues that will be associated with triggers for the local computer.
  4. Verify the computer has network connectivity.

The self-extracting setup program will install all DLL files into the \Winnt\System32 folder and all other files into the \Program Files\Microsoft\MSMQ Triggers folder of the local computer.

The MSMQ Triggers service can run under the LocalSystem account or under a domain administrative account. By default, this service runs under the LocalSystem account. The account used must be able to access queues to which triggers are associated with; specifically, the account must be granted the Log On As A Service right, and you must be granted the Peek Message permission for all applicable queues. For more information, see the Windows 2000 online documentation for Message Queuing.

MSMQ Triggers installs with a set of default configuration values that are stored in the registry. These values can be changed later using the MMC snap-in. For more information, see Changing Default Configuration Values.
